# 온라인 시험 플랫폼 서비스 명세서

## 1. 서비스 개요

### 1.1 핵심 기능
- **(Core)** 선생님(생성자)은 문제를 출제하여 '시험'을 만들고, 학생(풀이자)들은 이 '시험'에 응시할 수 있는 온라인 시험 플랫폼
- **(Teacher)** 선생님은 '그룹' 단위로 학생들을 관리하며, 학생들의 시험 성적을 테이블과 차트로 시각화하여 관리
- **(Scope)** 초등학생부터 대학생, 스터디 그룹 등 다양한 사용자가 그룹을 만들어 문제를 내고 풀 수 있음

### 1.2 사용자 역할
- **Teacher (문제 생성자)**: 그룹을 생성/관리하고, 시험을 출제하며, 학생 성적을 조회/분석
- **Student (문제 풀이자)**: 그룹에 가입하고, 시험에 응시하며, 자신의 성적을 확인
- **(Policy)** 회원가입 시 'Teacher' 또는 'Student' 역할을 선택해야 함 (역할 변경은 추후 관리자 승인 필요)

## 2. 사용자 스토리 및 화면 구성

### 2.1 공통 (홈, 인증)

#### 2.1.1 홈 화면 (로그아웃 상태)
**목적**: 서비스의 핵심 가치를 전달하고 신규 사용자를 유입시킴

**주요 구성요소**:
- 서비스의 핵심 기능을 설명하는 랜딩 페이지
- "문제 만들러 가기" (Teacher용) CTA 버튼
- "시험 응시하기" (Student용) CTA 버튼
- 로그인/회원가입 모달 버튼

**사용자 플로우**:
1. 사용자가 홈 화면 접속
2. 역할별 CTA 버튼 또는 로그인/회원가입 버튼 클릭
3. 해당 모달 또는 페이지로 이동

#### 2.1.2 회원가입 (모달)
**목적**: 신규 사용자 등록 및 역할 설정

**입력 필드**:
- 역할 선택: 'Teacher' / 'Student' 라디오 버튼 (필수)
- 로컬 회원가입:
  - 이메일 (필수, 유효성 검증)
  - 비밀번호 (필수, 최소 8자, 영문/숫자/특수문자 조합)
  - 비밀번호 확인 (필수)
  - 이름 (필수)
- SNS 회원가입:
  - 구글 로그인 버튼
  - 네이버 로그인 버튼
  - SNS 로그인 후에도 역할 선택 단계 필수

**정책**:
- 이메일 인증 절차를 거쳐야 회원가입이 완료됨
- 인증 이메일은 회원가입 후 자동 발송
- 인증 링크 유효 기간: 24시간

**유효성 검증**:
- 이메일 형식 검증 (example@domain.com)
- 비밀번호 강도 검증
- 비밀번호 일치 확인
- 중복 이메일 확인

**사용자 플로우**:
1. 역할 선택 (Teacher/Student)
2. 회원가입 방식 선택 (로컬/SNS)
3. 필수 정보 입력
4. 가입 완료 후 이메일 인증 안내
5. 이메일에서 인증 링크 클릭
6. 인증 완료 후 로그인 가능

#### 2.1.3 로그인 (모달)
**목적**: 기존 사용자 인증 및 서비스 접근

**로그인 방식**:
- 로컬 로그인:
  - 이메일 입력
  - 비밀번호 입력
- SNS 로그인:
  - 구글 로그인 버튼
  - 네이버 로그인 버튼

**추가 기능**:
- 비밀번호 찾기 (이메일로 재설정 링크 발송)
- 비밀번호 재설정 (임시 링크를 통한 새 비밀번호 설정)
- 자동 로그인 (선택 사항, 체크박스)

**에러 처리**:
- 이메일 미인증 계정 로그인 시도: "이메일 인증이 필요합니다" 안내 및 인증 메일 재발송 버튼
- 잘못된 이메일/비밀번호: "이메일 또는 비밀번호가 일치하지 않습니다" 메시지
- 계정 잠김 (5회 이상 로그인 실패): "계정이 일시적으로 잠겼습니다. 비밀번호 찾기를 이용해주세요" 메시지

### 2.2 Teacher (문제 생성자) 대시보드

#### 2.2.1 그룹 관리

##### 그룹 생성
**목적**: 학생들을 관리할 그룹 생성

**입력 필드**:
- 그룹 이름 (필수, 최대 50자)
- 그룹 설명 (선택, 최대 500자)
- 초대 코드 (자동 생성, 8자리 영문/숫자 조합)

**기능**:
- 그룹 생성 시 초대 코드 자동 생성
- 초대 코드는 고유해야 함 (중복 확인)

**사용자 플로우**:
1. "그룹 생성" 버튼 클릭
2. 그룹 정보 입력
3. 자동 생성된 초대 코드 확인
4. "생성" 버튼 클릭
5. 그룹 목록으로 이동

##### 그룹 목록
**목적**: 내가 생성했거나 속한 그룹 조회

**표시 정보**:
- 그룹 이름
- 멤버 수 (Teacher 수 / Student 수)
- 생성일
- 내 역할 (Owner, Co-Teacher)
- 활성 시험 수

**기능**:
- 그룹 검색 (이름으로 검색)
- 그룹 정렬 (최신순, 이름순, 멤버 수순)
- 그룹 카드 클릭 시 그룹 상세 페이지로 이동

##### 그룹 상세 (관리 페이지)
**목적**: 그룹 정보 수정 및 관리

**기본 정보 관리**:
- 그룹 이름 수정
- 그룹 설명 수정
- 그룹 삭제 (그룹 내 모든 데이터 삭제, 확인 모달 필수)

**초대 코드 관리**:
- 현재 초대 코드 표시
- 초대 코드 복사 버튼
- 초대 링크 복사 버튼 (`https://domain.com/join/{초대코드}`)
- 초대 코드 재발급 (기존 코드 무효화, 확인 모달 필수)
- 초대 코드 비활성화 (신규 가입 차단)

**정책**:
- 그룹 삭제 시 되돌릴 수 없음을 명확히 안내
- 그룹 Owner만 그룹 삭제 가능
- Co-Teacher는 멤버 관리, 시험 출제 가능하지만 그룹 삭제 불가

##### 그룹 인원 관리
**목적**: 그룹 멤버 조회 및 권한 관리

**멤버 목록 표시**:
- 테이블 형태로 표시:
  - 이름
  - 이메일
  - 역할 (Owner, Teacher, Student)
  - 가입일
  - 마지막 활동일
  - 작업 (권한 변경, 강제 퇴장)

**기능**:
- 멤버 검색 (이름, 이메일)
- 멤버 필터링 (역할별)
- 멤버 정렬 (이름순, 가입일순)
- 권한 변경: Student → Teacher 승격 (드롭다운 메뉴)
- 강제 퇴장: 멤버를 그룹에서 제거 (확인 모달 필수)

**정책**:
- Owner는 변경/삭제 불가
- Teacher는 Student만 관리 가능
- 강제 퇴장된 멤버는 초대 코드로 재가입 가능 (별도 차단 기능 없음)

#### 2.2.2 시험지 관리 (내가 낸 문제들)

##### 시험지 생성
**목적**: 새로운 시험지 생성

**입력 필드**:
- 시험지 이름 (필수, 최대 100자)
- 시험지 설명 (선택, 최대 1000자)
- 대상 그룹 선택 (필수, 다중 선택 가능)
- 응시 기간:
  - 시작일시 (필수, 날짜/시간 선택기)
  - 종료일시 (필수, 날짜/시간 선택기)
- 응시 제한 시간 (선택, 분 단위 입력, 예: 60분)
- 문제 순서 섞기 (셔플) 여부 (체크박스)
- 성적 공개 시점:
  - 즉시 공개 (제출 직후)
  - 시험 종료 후 공개
  - 수동 공개 (선생님이 직접 공개)
- 재응시 허용 여부 (체크박스, 허용 시 최고 점수/최근 점수 중 선택)

**유효성 검증**:
- 종료일시는 시작일시보다 이후여야 함
- 제한 시간은 1분 이상이어야 함 (설정 시)

**사용자 플로우**:
1. "시험지 생성" 버튼 클릭
2. 기본 정보 입력
3. 응시 설정 구성
4. "생성" 버튼 클릭
5. 문제 추가 페이지로 이동

##### 문제 추가/수정
**목적**: 시험지에 다양한 유형의 문제 추가 및 편집

**공통 기능**:
- 문제 유형 선택 (8가지 유형 중 선택)
- 문제 본문 작성 (리치 텍스트 에디터)
- 미디어 첨부:
  - 이미지 업로드 (JPG, PNG, GIF, 최대 5MB)
  - 동영상 URL 삽입 (YouTube, Vimeo 등)
  - 음성 파일 업로드 (MP3, WAV, 최대 10MB)
- 문제별 배점 설정 (필수, 숫자 입력)
- 문제 순서 변경 (드래그 앤 드롭)
- 문제 복사/삭제

**문제 목록 관리**:
- 추가된 문제 목록 표시 (순서, 유형, 본문 미리보기, 배점)
- 문제 검색 (본문 키워드)
- 문제 필터링 (유형별)
- 전체 배점 합계 표시

**사용자 플로우**:
1. 시험지 상세 페이지에서 "문제 추가" 버튼 클릭
2. 문제 유형 선택
3. 문제 내용 작성 (유형별 상세 정의 참조)
4. 미디어 첨부 (선택)
5. 배점 설정
6. "저장" 버튼 클릭
7. 문제 목록에 추가됨
8. 필요 시 추가 문제 계속 작성

##### 시험지 목록
**목적**: 생성한 시험지 조회 및 관리

**표시 정보**:
- 시험지 이름
- 대상 그룹
- 응시 기간
- 문제 수
- 총 배점
- 응시 인원 / 대상 인원
- 상태 (예정, 진행 중, 종료)

**기능**:
- 시험지 검색 (이름, 그룹)
- 시험지 필터링 (상태별)
- 시험지 복사 (동일한 구조로 새 시험지 생성)
- 시험지 수정
- 시험지 삭제 (응시 기록이 있으면 삭제 제한, 보관만 가능)

#### 2.2.3 성적 및 통계

##### 대시보드
**목적**: 그룹 및 시험 전반의 핵심 지표 요약

**표시 정보**:
- 전체 통계:
  - 총 그룹 수
  - 총 학생 수
  - 총 시험 수
  - 총 응시 건수
- 그룹별 요약:
  - 그룹 이름
  - 평균 점수
  - 응시율 (%)
- 시험별 요약:
  - 시험지 이름
  - 평균 점수
  - 응시 인원
  - 상태
- 최근 활동:
  - 최근 응시 내역 (학생 이름, 시험지, 점수, 응시 일시)

**시각화**:
- 월별 응시 추이 그래프 (라인 차트)
- 그룹별 평균 점수 비교 (바 차트)

##### 시험별 성적 테이블
**목적**: 특정 시험의 모든 학생 성적을 상세히 조회

**기본 컬럼**:
- 학생 이름
- 이메일
- 점수 (자동 채점 점수 / 총점)
- 응시 일시
- 소요 시간 (분:초)
- 제출 상태 (제출 완료, 시간 초과, 미응시)

**커스텀 컬럼 추가**:
- 최대 5개의 추가 컬럼 생성 가능
- 예: 출결, 수행평가 1, 수행평가 2, 과제, 기타
- 각 컬럼에 점수 직접 입력 (숫자)
- 합산 점수 자동 계산 (시험 점수 + 커스텀 컬럼 점수들)

**테이블 기능**:
- 정렬 (Sort): 모든 컬럼 기준 오름차순/내림차순
- 필터링 (Filter): 점수 범위, 제출 상태, 특정 학생
- 검색: 학생 이름, 이메일
- CSV 내보내기
- Excel 내보내기

**사용자 플로우**:
1. 시험지 목록에서 특정 시험 선택
2. "성적 보기" 탭 클릭
3. 성적 테이블 조회
4. 필요 시 커스텀 컬럼 추가 ("컬럼 추가" 버튼)
5. 커스텀 컬럼에 점수 입력
6. 필요 시 정렬/필터링/검색
7. "내보내기" 버튼으로 CSV/Excel 다운로드

##### 시험별 통계 (시각화)
**목적**: 시험 결과를 시각적으로 분석

**제공 통계**:
1. **기술 통계**:
   - 평균 (Mean)
   - 중앙값 (Median)
   - 최솟값 (Min)
   - 최댓값 (Max)
   - 표준편차 (Standard Deviation)
   - 분산 (Variance)

2. **점수 분포 히스토그램**:
   - X축: 점수 구간 (10점 단위)
   - Y축: 학생 수
   - 색상 코딩: 구간별 색상 다르게 표시

3. **백분위 정보**:
   - 25백분위, 50백분위(중앙값), 75백분위 표시

**시각화 도구**:
- 히스토그램 (점수 분포)
- 박스 플롯 (Box Plot) - 사분위수 표시
- 정규 분포 곡선 오버레이 (선택 사항)

##### 문제별 통계 (시각화)
**목적**: 각 문제의 난이도 및 학생들의 이해도 파악

**제공 통계**:
1. **문제별 정답률**:
   - 문제 번호 / 유형
   - 정답률 (%)
   - 정답자 수 / 전체 응시자 수

2. **객관식 선택지 분석**:
   - 각 선택지별 선택 비율 (%)
   - 가장 많이 선택한 오답 표시 (색상 강조)
   - 파이 차트 또는 바 차트로 시각화

3. **난이도 분류**:
   - 정답률 80% 이상: 쉬움 (초록색)
   - 정답률 50-80%: 보통 (노란색)
   - 정답률 50% 미만: 어려움 (빨간색)

**시각화 도구**:
- 바 차트 (문제별 정답률)
- 파이 차트 (선택지별 선택 비율)
- 히트맵 (문제 × 학생 정오답 매트릭스)

##### 학생 개인 통계
**목적**: 특정 학생의 학습 성취도 및 강약점 분석

**제공 정보**:
1. **누적 성적 그래프**:
   - X축: 시험 일시 (시간순)
   - Y축: 점수 (%)
   - 라인 차트로 점수 추이 표시
   - 평균선 표시 (그룹 평균과 비교)

2. **강점 / 약점 분석**:
   - 문제 유형별 정답률 (%)
     - 객관식: XX%
     - O/X: XX%
     - 주관식(단답): XX%
     - 서술형: XX%
     - 순서 배열: XX%
     - 연결하기: XX%
     - 빈칸 채우기: XX%
     - 표 완성: XX%
   - 레이더 차트로 시각화
   - 정답률 높은 유형 (강점) 표시
   - 정답률 낮은 유형 (약점) 표시 및 개선 제안

3. **전체 요약**:
   - 총 응시 시험 수
   - 전체 평균 점수
   - 그룹 내 랭킹 (선생님 공개 설정 시)

**사용자 플로우**:

2. 학생 개인 통계 페이지로 이동
3. 누적 성적, 유형별 정답률, 전체 요약 확인
4. 필요 시 PDF로 내보내기

#### 2.2.4 채점 관리 (서술형)

**채점 대기 큐**:

  - 시험지 이름

  - 학생 이름
  - 제출 일시

  - 채점 상태 (대기 중, 채점 완료)

**채점 화면**:

- 학생 답안 표시 (텍스트 영역)
- 점수 입력 필드 (0 ~ 문제 배점)
- 피드백 입력 필드 (선택, 학생에게 전달될 코멘트)

**기능**:
- 채점 필터링 (시험지별, 학생별, 채점 상태별)
- 채점 정렬 (제출일 순, 학생 이름 순)
- 일괄 채점 (동일 점수 부여, 체크박스로 선택)

**정책**:
- 부분 점수 허용 (0 ~ 문제 배점 범위 내)
- 채점 완료 후 수정 가능
- 학생은 채점 완료 전까지 "채점 중" 상태로 표시됨

**사용자 플로우**:
1. "채점 관리" 메뉴 클릭
2. 채점 대기 목록 확인
3. 특정 답안 클릭하여 채점 화면 진입
4. 학생 답안 읽기
5. 점수 입력 및 피드백 작성 (선택)
6. "저장 및 다음" 버튼 클릭
7. 다음 답안으로 자동 이동
8. 모든 채점 완료 시 대기 큐에서 제거

## 3. 문제 타입 상세 정의

### 3.1 공통 사항
- **미디어 첨부**: 모든 문제 타입은 문제 본문에 이미지, 동영상(URL), 음성 파일을 첨부할 수 있음
- **개별 배점**: 문제별로 배점(점수)을 개별 설정할 수 있어야 함
- **문제 번호**: 시험지 내에서 자동으로 순번 부여 (1번, 2번, ...)

---

### 3.2 문제 유형별 상세

#### 3.2.1 객관식
**설명**: 여러 선택지 중 정답을 선택하는 문제

**출제 설정**:
- 선택지 개수: 최소 2개, 최대 10개
- 각 선택지 입력: 텍스트 또는 이미지
- 정답 설정: 단일 정답 또는 복수 정답 (체크박스로 선택)
- 복수 정답 시 부분 점수 설정 가능 (예: 3개 정답 중 2개 맞으면 66%)

**응시 UI**:
- 단일 정답: 라디오 버튼
- 복수 정답: 체크박스 (안내 문구: "복수 정답 문제입니다")

**채점**:
- 자동 채점
- 단일 정답: 정답 선택 시 100%, 오답 시 0%
- 복수 정답: 정답 개수 대비 맞힌 개수 비율로 부분 점수 (설정 시)

**예시**:
```
문제: 다음 중 포유류에 해당하는 동물을 모두 고르세요. (복수 정답)
① 고래 ✓
② 펭귄
③ 박쥐 ✓
④ 상어
⑤ 돌고래 ✓

정답: ①, ③, ⑤
배점: 10점
부분 점수: 2개 맞으면 6.7점, 1개 맞으면 3.3점
```

#### 3.2.3 주관식 (단답형)

- 문제 본문 입력
- 정답 입력: 텍스트 입력 (여러 개 가능)
- 복수 정답 설정 가능 (예: '컴퓨터', 'computer', 'Computer' 모두 정답)
- 대소문자 구분 여부 체크박스

- 텍스트 입력란 (한 줄)
- 플레이스홀더: "답을 입력하세요"

**채점**:
- 자동 채점
- 입력한 답안이 정답 목록 중 하나와 일치하면 100%
- 일치하지 않으면 0%
- 대소문자 및 공백 처리는 출제 설정에 따름

**예시**:
```
문제: 대한민국의 수도는 어디인가요?

배점: 5점
```

#### 3.2.5 순서 배열형
**설명**: 주어진 보기들을 올바른 순서대로 배열하는 문제

**출제 설정**:
- 보기 추가 (텍스트 또는 이미지, 최소 2개)
- 올바른 순서 설정 (드래그 앤 드롭으로 정렬)
- 부분 점수 설정:
  - 전체 일치만 정답: 순서가 완전히 일치해야 100%
  - 부분 점수 허용: 올바른 위치의 보기 개수 비율로 점수 부여

**응시 UI**:
- 보기 목록 (초기에는 랜덤 순서 또는 출제 순서)
- 드래그 앤 드롭으로 순서 변경
- 순서 표시 (1, 2, 3, ...)

**채점**:
- 자동 채점
- 전체 일치: 모든 보기가 올바른 위치에 있으면 100%
- 부분 점수 (설정 시): (올바른 위치의 보기 수 / 전체 보기 수) × 100%

**예시**:
```
문제: 다음 역사적 사건을 시간 순서대로 배열하시오.
보기:
- 6.25 전쟁
- 광복
- 88 올림픽
- 한글 창제

정답 순서:
1. 한글 창제
2. 광복
3. 6.25 전쟁
4. 88 올림픽

배점: 10점
부분 점수: 2개 위치 맞으면 5점, 3개 맞으면 7.5점
```

#### 3.2.6 연결하기 / 짝짓기
**설명**: 왼쪽과 오른쪽 보기를 선으로 연결하거나 드래그 앤 드롭으로 짝짓는 문제

**출제 설정**:
- 왼쪽 보기 추가 (텍스트 또는 이미지, 최소 2개)
- 오른쪽 보기 추가 (텍스트 또는 이미지, 최소 2개)
- 올바른 짝 설정 (왼쪽 보기 A → 오른쪽 보기 B)
- 부분 점수 설정:
  - 전체 일치만 정답: 모든 짝이 일치해야 100%
  - 부분 점수 허용: 올바른 짝의 개수 비율로 점수 부여

**응시 UI**:
- 방식 1 (선 연결):
  - 왼쪽 보기 목록
  - 오른쪽 보기 목록
  - 왼쪽 항목 클릭 → 오른쪽 항목 클릭 → 선으로 연결
  - 연결 해제 버튼
- 방식 2 (드래그 앤 드롭):
  - 왼쪽 보기는 고정
  - 오른쪽 보기를 드래그하여 왼쪽 보기 옆 슬롯에 드롭

**채점**:
- 자동 채점
- 전체 일치: 모든 짝이 올바르면 100%
- 부분 점수 (설정 시): (올바른 짝 수 / 전체 짝 수) × 100%

**예시**:
```
문제: 다음 나라와 수도를 연결하시오.

왼쪽 (나라):        오른쪽 (수도):
- 대한민국          - 도쿄
- 일본              - 베이징
- 중국              - 서울

정답:
- 대한민국 ─ 서울
- 일본 ─ 도쿄
- 중국 ─ 베이징

배점: 12점
부분 점수: 1쌍 맞으면 4점, 2쌍 맞으면 8점
```

#### 3.2.7 빈칸 채우기형

### 4.1 주요 엔티티

#### User (사용자)
- id (PK)
- email (UK)

- name
- role (Teacher, Student)
- emailVerified (boolean)
- createdAt
- updatedAt

#### Group (그룹)
- id (PK)
- name
- description
- inviteCode (UK)
- inviteCodeActive (boolean)
- ownerId (FK → User)
- createdAt
- updatedAt

#### GroupMember (그룹 멤버)
- id (PK)
- groupId (FK → Group)
- userId (FK → User)
- role (Owner, Teacher, Student)
- joinedAt

#### Exam (시험지)
- id (PK)
- name
- description
- creatorId (FK → User)
- startDate
- endDate
- timeLimit (분)
- shuffleQuestions (boolean)
- scoreReleaseType (immediate, after_end, manual)
- allowRetake (boolean)
- retakeScoreType (highest, latest)
- createdAt
- updatedAt

#### ExamGroup (시험-그룹 매핑, 다대다)
- id (PK)
- examId (FK → Exam)
- groupId (FK → Group)

#### Question (문제)
- id (PK)
- examId (FK → Exam)
- type (객관식, OX, 단답형, 서술형, 순서배열, 짝짓기, 빈칸채우기, 표완성)
- order (순서)
- content (문제 본문, JSON 또는 HTML)
- mediaUrls (JSON: 이미지, 동영상, 음성 URL 배열)
- points (배점)
- answerData (JSON: 유형별 정답 및 설정 데이터)
- createdAt
- updatedAt

#### Submission (시험 제출)
- id (PK)
- examId (FK → Exam)
- userId (FK → User)
- startedAt
- submittedAt
- timeSpent (초)
- status (in_progress, submitted, timeout)
- createdAt

#### Answer (답안)
- id (PK)
- submissionId (FK → Submission)
- questionId (FK → Question)
- answerData (JSON: 유형별 학생 답안 데이터)
- score (획득 점수)
- isCorrect (boolean, nullable)
- feedback (서술형 피드백)
- gradedAt (채점 일시)
- gradedBy (FK → User, 채점자)

#### CustomColumn (커스텀 컬럼)
- id (PK)
- examId (FK → Exam)
- name (예: 출결, 수행평가 1)
- maxScore (만점)
- order (컬럼 순서)

#### CustomScore (커스텀 점수)
- id (PK)
- customColumnId (FK → CustomColumn)
- userId (FK → User)
- score (점수)

### 4.2 관계
- User ↔ Group: 다대다 (GroupMember 중간 테이블)
- User → Exam: 일대다 (Creator)
- Exam ↔ Group: 다대다 (ExamGroup 중간 테이블)
- Exam → Question: 일대다
- User + Exam → Submission: 다대다 (Submission 중간 테이블)
- Submission → Answer: 일대다
- Exam → CustomColumn: 일대다
- CustomColumn + User → CustomScore: 다대다

## 7. 보안 및 정책

### 7.1 보안
- **비밀번호**: bcrypt 또는 Argon2로 해싱
- **JWT**: HttpOnly 쿠키로 저장, Refresh Token 사용
- **CSRF 방지**: CSRF 토큰 사용
- **XSS 방지**: 사용자 입력 Sanitize (DOMPurify)
- **SQL Injection 방지**: ORM 사용, Prepared Statement
- **파일 업로드 검증**: MIME 타입 확인, 파일 크기 제한
- **Rate Limiting**: API 요청 제한 (DDoS 방지)

### 7.2 개인정보 보호
- **GDPR/개인정보보호법 준수**: 데이터 수집 동의, 삭제 요청 처리
- **데이터 암호화**: 전송 시 HTTPS, 저장 시 중요 데이터 암호화
- **로그 관리**: 개인정보 로그 최소화, 민감 정보 마스킹

### 7.3 시험 부정행위 방지
- **페이지 이탈 감지**: 시험 중 다른 탭/창 이동 시 경고 및 로그 기록
- **랜덤 문제 순서**: 학생마다 문제 순서 다르게 출제 (설정 시)
- **시간 제한 엄격 적용**: 서버 시간 기준으로 타임아웃
- **IP 로깅**: 제출 시 IP 주소 기록 (선택 사항, 개인정보 정책 고려)
- **복사 방지**: 우클릭 방지, 복사 방지 (선택 사항, UX 고려)

### 8.1 디자인 철학
**"귀엽고 친근한, 학습이 즐거워지는 디자인"**

학생들이 시험이라는 부담스러운 경험을 더욱 편안하고 즐겁게 느낄 수 있도록 귀여운 일러스트와 부드러운 색감을 사용합니다. 초등학생부터 대학생까지 모두가 친근하게 느낄 수 있는 디자인을 지향합니다.

### 8.2 컬러 팔레트

#### 주요 컬러 (Primary Colors)
- **주 색상 (Primary)**: `#FF6B9D` (부드러운 핑크) - CTA 버튼, 중요 액션
- **보조 색상 (Secondary)**: `#FFA07A` (코랄 오렌지) - 보조 버튼, 강조
- **강조 색상 (Accent)**: `#FFD93D` (밝은 노란색) - 배지, 알림, 하이라이트

#### 중립 컬러 (Neutral Colors)
- **배경색 (Background)**: `#FAFBFC` (매우 밝은 회색) - 페이지 배경
- **카드 배경 (Card)**: `#FFFFFF` (화이트) - 카드, 모달
- **텍스트 (Text Primary)**: `#2D3748` (진한 회색) - 본문 텍스트
- **텍스트 (Text Secondary)**: `#718096` (중간 회색) - 보조 텍스트

#### 상태 컬러 (Status Colors)
- **성공 (Success)**: `#68D391` (밝은 초록) - 정답, 성공 메시지
- **에러 (Error)**: `#FC8181` (부드러운 빨강) - 오답, 에러 메시지
- **경고 (Warning)**: `#F6AD55` (따뜻한 오렌지) - 경고, 주의 필요
- **정보 (Info)**: `#63B3ED` (하늘색) - 정보 메시지, 안내

#### 그라데이션 (Gradients)
- **메인 그라데이션**: `linear-gradient(135deg, #FF6B9D 0%, #FFA07A 100%)`
- **배경 그라데이션**: `linear-gradient(180deg, #FFF5F7 0%, #FAFBFC 100%)`
- **버튼 호버**: `linear-gradient(135deg, #FF5A8F 0%, #FF956C 100%)`

### 8.3 타이포그래피

#### 폰트 패밀리
- **한글**: Pretendard Variable (가변 폰트) - 부드럽고 가독성 좋은 한글 폰트
- **영문/숫자**: Inter Variable - 모던하고 깔끔한 영문 폰트
- **코드/고정폭**: JetBrains Mono - 코드 블록용

#### 폰트 크기 체계
```
- Heading 1 (h1): 2.5rem (40px) / Bold / Line-height: 1.2
- Heading 2 (h2): 2rem (32px) / Bold / Line-height: 1.3
- Heading 3 (h3): 1.5rem (24px) / SemiBold / Line-height: 1.4
- Heading 4 (h4): 1.25rem (20px) / SemiBold / Line-height: 1.4
- Body Large: 1.125rem (18px) / Regular / Line-height: 1.6
- Body: 1rem (16px) / Regular / Line-height: 1.6
- Body Small: 0.875rem (14px) / Regular / Line-height: 1.5
- Caption: 0.75rem (12px) / Regular / Line-height: 1.4
```

### 8.4 일러스트 및 아이콘

#### 일러스트 스타일
- **스타일**: 플랫 디자인, 부드러운 곡선, 파스텔 톤
- **캐릭터**: 귀여운 동물 캐릭터 (팬더, 고양이, 강아지 등)
  - 선생님 = 안경 쓴 올빼미 캐릭터 (지혜의 상징)
  - 학생 = 책가방 멘 토끼 캐릭터 (귀여움, 활발함)
  - 성공/정답 = 웃는 별 캐릭터
  - 응원 = 깃발 든 치타 캐릭터

#### 주요 일러스트 사용 위치
1. **랜딩 페이지**:
   - 히어로 섹션: 선생님과 학생 캐릭터가 함께 공부하는 장면
   - 기능 설명: 각 기능별 캐릭터 일러스트 (시험 만들기, 응시하기, 성적 보기)

2. **빈 상태 (Empty State)**:
   - 그룹이 없을 때: 토끼가 초대장을 기다리는 일러스트
   - 시험이 없을 때: 올빼미가 책을 펼치는 일러스트
   - 아직 응시 안 했을 때: 토끼가 연필을 들고 있는 일러스트

3. **상태 피드백**:
   - 시험 제출 완료: 별 캐릭터가 박수치는 애니메이션
   - 로딩 중: 토끼가 점프하는 애니메이션
   - 에러 발생: 당황한 팬더 일러스트

4. **성적 페이지**:
   - 100점: 트로피 든 캐릭터 + 폭죽 애니메이션
   - 80점 이상: 웃는 별 캐릭터
   - 향상됨: 위로 날아가는 풍선 캐릭터

#### 아이콘 스타일
- **스타일**: Rounded, Outlined (선 두께: 2px)
- **라이브러리**: Lucide Icons (React) - 부드럽고 깔끔한 아이콘
- **색상**: 기본은 회색 (`#718096`), 활성화 시 Primary 색상 적용
- **크기**: 16px (small), 20px (medium), 24px (large), 32px (extra large)

### 8.7 애니메이션 및 인터랙션

#### 애니메이션 원칙
- **Easing**: `cubic-bezier(0.4, 0, 0.2, 1)` (부드러운 가속/감속)
- **Duration**:
  - 빠른 애니메이션: 150ms (호버, 클릭)
  - 일반 애니메이션: 300ms (모달, 드롭다운)
  - 느린 애니메이션: 500ms (페이지 전환)

#### 주요 인터랙션 효과
1. **버튼 클릭**: 살짝 눌리는 효과 (scale: 0.98)
2. **카드 호버**: 위로 올라가고 그림자 확대
3. **입력 필드 포커스**: Border 색상 변경 + 부드러운 그림자
4. **로딩**: 귀여운 캐릭터가 점프하는 애니메이션
5. **성공 피드백**: Confetti 애니메이션 (100점 시)
6. **에러 피드백**: 좌우로 흔들리는 애니메이션 (Shake)

#### 마이크로 인터랙션
- **체크박스 선택**: 체크 표시가 그려지는 애니메이션
- **토글 스위치**: 부드럽게 슬라이드
- **별점**: 호버 시 별이 반짝이는 효과
- **프로그레스 바**: 채워지는 애니메이션 (트랜지션)

### 8.8 특별 화면별 디자인 가이드

#### 랜딩 페이지
- **히어로 섹션**:
  - 큰 헤드라인 + 캐릭터 일러스트 (오른쪽)
  - 그라데이션 배경 (부드러운 핑크 → 화이트)
  - CTA 버튼 2개 (Teacher용, Student용) - 서로 다른 색상

- **기능 소개 섹션**:
  - 3-4개 카드, 각 카드마다 캐릭터 일러스트
  - 아이콘 + 제목 + 설명
  - 호버 시 카드가 살짝 커지는 효과

#### 시험 응시 화면
- **타이머**: 큰 원형 프로그레스 바 (시계 캐릭터 중앙)
  - 남은 시간이 5분 미만이면 빨간색으로 변경 + 깜빡임
- **문제 카드**: 흰색 카드, 넓은 패딩, 부드러운 그림자
- **문제 번호 네비게이션**: 동그란 버튼들 (답안 작성 시 파란색)
- **제출 확인 모달**: 귀여운 캐릭터가 "정말 제출할까요?" 물어보는 일러스트

#### 성적 확인 화면
- **점수 카드**: 큰 숫자 + 캐릭터 일러스트
  - 90-100점: 트로피 + 황금색 테두리
  - 80-89점: 별 캐릭터 + 은색 테두리
  - 70-79점: 웃는 얼굴 + 청동색 테두리
  - 70점 미만: 응원하는 캐릭터 + 회색 테두리
- **정답/오답 표시**: 큰 O, X 아이콘 (둥근 배경)
- **차트**: 파스텔 톤 색상, 부드러운 곡선

#### 빈 상태 (Empty State)
- 모든 빈 상태에 귀여운 일러스트 추가
- 메시지: 친근하고 격려하는 톤 ("아직 시험이 없어요! 첫 시험을 만들어볼까요?")
- CTA 버튼: 명확한 다음 액션 ("시험 만들기")

### 8.10 접근성 (Accessibility)

- **색상 대비**: WCAG AA 기준 최소 4.5:1 (본문 텍스트)
- **포커스 표시**: 키보드 포커스 시 명확한 아웃라인 (2px solid #FF6B9D)
- **대체 텍스트**: 모든 일러스트/아이콘에 의미 있는 alt 텍스트
- **터치 타겟**: 최소 44x44px (모바일)
- **폰트 크기**: 최소 16px (본문)

## 9. 성공 지표 (KPI)

### 9.1 사용자 관련
- **가입자 수**: 월별 신규 가입자 추이
- **활성 사용자 수 (DAU/MAU)**: 일별/월별 활성 사용자
- **사용자 유지율**: 월별 재방문율

### 9.2 플랫폼 사용
- **생성된 그룹 수**: 총 그룹 수, 월별 신규 그룹
- **생성된 시험 수**: 총 시험 수, 월별 신규 시험
- **응시 건수**: 총 응시 건수, 월별 응시
- **평균 시험당 응시 인원**: 시험 활용도

### 9.3 품질
- **시험 완료율**: 시작한 시험 중 제출 완료한 비율
- **평균 응시 시간**: 시험 유형별 평균 소요 시간
- **오류율**: 시스템 에러 발생 빈도

### 9.4 만족도
- **NPS (Net Promoter Score)**: 서비스 추천 의향
- **사용자 피드백**: 만족도 설문 점수
- **재방문율**: 한 번 사용한 사용자의 재사용 비율
